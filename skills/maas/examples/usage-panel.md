# 用量面板

本示例演示如何组合多个接口，为用户呈现一个完整的 AI 用量分析面板。

---

## 面板目标

当用户说"帮我看下最近的 API 用量情况"或"展示一下本月用量面板"时，输出以下内容：

1. **本月 Token 消耗趋势**（按天粒度时间序列，按模型分组）
2. **各 API Key 用量占比**（本月各 Key 的 token 消耗对比）
3. **各 Key 的配额使用率**（日限额 / 月限额已用百分比）

---

## 数据获取步骤

### Step 1：获取本月 Token 消耗趋势

```
GET /ai/inapi/v3/stat/new
  ?start=2026-04-01T00:00:00+08:00
  &end=2026-04-30T23:59:59+08:00
  &g=day
```

**响应结构：**
```json
{
  "status": true,
  "data": [
    {
      "name": "qwen-turbo",
      "items": [
        {
          "name": "文本输入 tokens",
          "unit": "k/tokens",
          "total": 1234.5,
          "values": [
            { "time": "2026-04-01T00:00:00+08:00", "value": 45.2 },
            { "time": "2026-04-02T00:00:00+08:00", "value": 88.1 }
          ]
        },
        {
          "name": "文本输出 tokens",
          "unit": "k/tokens",
          "total": 312.0,
          "values": [
            { "time": "2026-04-01T00:00:00+08:00", "value": 12.0 },
            { "time": "2026-04-02T00:00:00+08:00", "value": 20.5 }
          ]
        }
      ]
    }
  ]
}
```

**说明：**
- `data[].name`：模型名称
- `data[].items[].name`：计费项名称（输入/输出/缓存等）
- `data[].items[].total`：查询范围内累计量（单位由 `unit` 字段指定）
- `data[].items[].values[]`：按 `g` 粒度的分时数据，`time` 为区间左端点

---

### Step 2：获取各 API Key 的用量与配额使用率

```
GET /ai/inapi/v3/apikeys
```

**响应结构（核心字段）：**
```json
{
  "data": [
    {
      "key": "sk-ab***c123",
      "name": "生产环境",
      "enabled": true,
      "totalTokens": 5200000,
      "quota": {
        "daily": {
          "used": 120000,
          "limit": 500000
        },
        "monthly": {
          "used": 3800000,
          "limit": 5000000
        }
      }
    }
  ]
}
```

**说明：**
- `totalTokens`：累计消耗的 token 总数
- `quota.daily.used / limit`：今日已用 / 日限额（`-1` 表示未设限）
- `quota.monthly.used / limit`：本月已用 / 月限额（`-1` 表示未设限）

---

### Step 3（可选）：计费项名称映射

若需将用量数据中的 `key` 字段（如 `input`、`output`、`cache_creation`）转换为人类可读名称：

```
GET /ai/inapi/v3/market/pricingitems
```

## 注意事项

- `/ai/inapi/v3/stat/new` 接口对时间范围没有严格限制说明，但建议不超过 35 天
- 不同模型的计费项名称可能不同（如图像模型无 `input`/`output` 概念），需根据实际 `name` 字段展示
- 配额字段 `limit = -1` 表示未设置限额，展示为"未设限"而非"-1%"
- 时序数据中某天无数据时，该天不会出现在 `values` 数组里，展示时需补 0
